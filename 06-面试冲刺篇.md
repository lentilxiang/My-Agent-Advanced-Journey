## 项目深挖准备（每个项目 8 问）

1. **Why** — 为什么做？解决什么问题？
2. **What** — 整体架构？核心模块？
3. **How** — 关键技术怎么实现的？
4. **Metrics** — 效果怎么衡量？具体数据？
5. **Challenge** — 最大挑战？怎么解决的？
6. **Trade-off** — 技术选型？为什么？
7. **Improvement** — 重新做有什么改进？
8. **Scale** — 如何应对流量增长？瓶颈在哪？

## 系统设计题

### 常见题目

1. **设计企业级 RAG 系统** — 文档处理、检索优化、高可用、增量更新
2. **设计多 Agent 客服系统** — Agent 路由、知识库集成、人工转接
3. **设计代码生成 Agent** — 代码理解、测试生成、安全沙箱
4. **设计 LLM 推理服务** — 高可用、负载均衡、模型更新、成本优化

### 答题框架

```
1. 需求澄清（5min）——功能/非功能需求、规模、延迟、可用性
2. 高层架构（10min）——核心组件、数据流、技术选型
3. 核心模块深入（15min）——重点模块详细设计
4. 扩展性和优化（5-10min）——性能、扩展、容灾
```

## 面试话术

### 不会的题怎么答

> "这个问题我没有深入研究过，但我可以分享一下理解和思路：
> 1. 从 xxx 原理出发，我认为...
> 2. 类似的问题我有经验，可以类比...
> 3. 如果让我去解决，我会先...然后...
> 比起不确定的答案，我更愿意承认盲区并展示思考过程。"

### 展示深度思考

- 说"为什么"而非只是"是什么"
- 主动讲 trade-off
- 对比不同方案并说明选择理由
- 关联实际项目经验
- 提及最新技术进展
