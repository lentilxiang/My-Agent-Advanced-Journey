## 第3周：Prompt Engineering → Context Engineering

### 核心技巧

| 技巧 | 说明 | 适用场景 |
|------|------|---------|
| Zero-shot | 直接给指令 | 简单任务 |
| Few-shot | 提供 3-5 个示例 | 格式控制 |
| Chain-of-Thought | "Let's think step by step" | 推理任务 |
| ReAct | 思考-行动-观察循环 | Agent 基础 |
| Self-Consistency | 多次采样投票 | 提高准确率 |
| Tree of Thoughts | 树状搜索 | 复杂规划 |

### Context Engineering（2025 关键转变）

不只是写好 prompt，而是**系统设计模型接收的全部上下文**：

- 上下文组装（Context Assembly）
- 上下文压缩（Context Compression）
- 上下文排序（Context Ordering）——Lost in the Middle
- 上下文缓存（Context Caching）
- 动态上下文管理

## 第4-5周：RAG 完全掌握

> RAG 是面试必考、工作必用的核心技术。

### Naive RAG 完整流程

```
用户查询 → 查询向量化 → 向量检索 → 获取文档片段 → 组装上下文 → LLM生成答案
```

### 关键环节

#### 文档处理
- 文本分割策略：固定大小 / 递归字符 / 语义分割 / 按结构分割
- Chunk Size：通常 256-1024 tokens
- Chunk Overlap：10-20%

#### Embedding 模型
- OpenAI text-embedding-3
- BGE 系列（中文最佳）
- GTE 系列（阿里）
- 评估基准：MTEB、C-MTEB

#### 向量数据库

| 数据库 | 特点 | 场景 |
|--------|------|------|
| Chroma | 轻量级 | 本地开发 |
| Milvus | 分布式 | 生产环境 |
| Qdrant | Rust 实现 | 高性能 |
| FAISS | Meta 开源 | 基础设施 |

### Advanced RAG 优化

- **混合检索**：向量 + BM25 + RRF 融合
- **查询改写**：Multi-Query、HyDE
- **重排序**：Cross-Encoder Reranker
- **幻觉检测**：NLI-based 方法
- **评估框架**：RAGAs（Faithfulness、Relevance、Precision、Recall）

### 高级 RAG 架构

- GraphRAG：知识图谱 + RAG
- Agentic RAG：Agent 驱动的自适应检索
- Self-RAG：模型自主判断是否检索

## 第5-6周：Agent 设计模式

### Agent 核心组件

```
┌──────────────────────────────────────┐
│            AI Agent                   │
├──────────┬──────────┬────────────────┤
│  规划    │  记忆    │  工具          │
│ Planning │ Memory   │ Tools          │
├──────────┼──────────┼────────────────┤
│ 任务分解 │ 短期记忆 │ API 调用       │
│ 推理链   │ 长期记忆 │ 代码执行       │
│ 动态调整 │ 工作记忆 │ 搜索           │
└──────────┴──────────┴────────────────┘
```

### 核心设计模式

#### 1. ReAct（Reasoning + Acting）
```
Thought → Action → Observation → Thought → ... → Final Answer
```

#### 2. Plan-and-Execute
```
1. 规划阶段：分解为子任务列表
2. 执行阶段：按序执行，可选动态调整
```

#### 3. Reflection
```
生成 → 反思/批评 → 修改 → 再反思 → 满意输出
```

#### 4. Tool Use / Function Calling
- 模型输出结构化工具调用
- 参数校验和错误处理
- 工具选择策略

### 多 Agent 协作模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| Supervisor | 主 Agent 协调子 Agent | 任务分解明确 |
| Hierarchical | 多层 Agent 组织 | 大型复杂系统 |
| Debate | 多角度讨论达成共识 | 多视角分析 |
| Swarm | 去中心化群体协作 | 灵活协作 |

### Agent 记忆系统

- **短期记忆**：对话上下文、工作区状态
- **长期记忆**：向量数据库存储历史交互、摘要记忆
- **工作记忆**：临时推理空间
