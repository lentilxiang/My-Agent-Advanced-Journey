## 第7周：LangGraph 精通

### LangChain 核心概念

- LCEL（LangChain Expression Language）
- Chat Models / Prompt Templates / Output Parsers
- Tools / Retrievers / Chains / Memory

### LangGraph —— Agent 编排框架（重点）

| 概念 | 说明 |
|------|------|
| State | 图的状态定义（TypedDict 或 Pydantic） |
| Node | 处理函数（每个节点执行一个操作） |
| Edge | 节点间连接（条件边、普通边） |
| Graph | 有向图，定义整个工作流 |

### 关键特性

- 循环支持（区别于 DAG）
- Human-in-the-loop（人工介入）
- Checkpointing（持久化状态）
- Streaming（流式输出）
- Subgraph（子图）

### 常用模式实现

- ReAct Agent
- Supervisor Multi-Agent
- Plan-and-Execute
- Human-in-the-loop 审核流

## 第8周：其他框架 + MCP + A2A

### CrewAI

- 基于角色的多 Agent 框架
- 定义 Agent（角色、目标、工具）+ Task
- 适合快速搭建多角色协作系统
- GitHub: https://github.com/crewAIInc/crewAI

### Dify

- 低代码 AI 应用开发平台
- 可视化编排工作流
- 内置 RAG、Agent、工具集成
- GitHub: https://github.com/langgenius/dify

### AutoGen（微软）

- 对话驱动的多 Agent 框架
- 支持人类参与对话
- 适合研究和实验

### MCP（Model Context Protocol）

> Anthropic 发起的工具集成协议，标准化 LLM 与外部工具的通信。

架构：
```
Host（宿主应用）
  └── Client（MCP 客户端）
        └── Server（MCP 服务端）
              ├── Tools（可调用函数）
              ├── Resources（可读取数据）
              └── Prompts（预定义模板）
```

- 通信协议：JSON-RPC 2.0
- 传输方式：stdio（本地）+ SSE over HTTP（远程）

### A2A（Agent-to-Agent Protocol）

> Google 发起的 Agent 间通信协议

- Agent Card：能力描述
- Task Negotiation：任务协商
- Context Passing：结构化上下文传递

### 框架选型建议

| 场景 | 推荐框架 |
|------|---------|
| 生产级 Agent 系统 | LangGraph |
| 快速原型验证 | Dify / CrewAI |
| 研究/实验 | AutoGen / 纯代码 |
| 多角色协作演示 | CrewAI |
| 工具集成 | MCP + LangChain |
